All Aegis clients share a single cryptographic implementation in **aegis-core** (Rust). Native clients (iOS, macOS, Android) call it via the **aegis-ffi** UniFFI bridge. The web client uses **@noble/post-quantum** — pure TypeScript implementations of ML-KEM-768 and ML-DSA-65 that mirror the Rust library behavior exactly.

---

## Web client (aegis-web)

**Stack:** React 18 + Vite + TypeScript + Tailwind CSS
**Deployment:** Cloudflare Workers (edge-cached SPA)
**Crypto:** @noble/ciphers, @noble/curves, @noble/hashes, @noble/post-quantum (pure JS — no WASM)

The web client is a full-featured browser SPA. No server-side rendering; all crypto runs in the browser.

**Features:**
- **Setup flow** — create a new identity or import an existing one
- **Identity management** — view keys, publish to relay, replenish prekey pool
- **Compose** — recipient resolution (alias or identity ID), suite auto-detection, one-time prekey claiming, hybrid PQ encryption, dual signature, push to relay
- **Inbox** — fetch envelopes from relay, display threads, open/decrypt messages, consume and delete local prekey secrets after decryption
- **Passkey integration** — WebAuthn credential creation and verification for vault unlock
- **Encrypted vault** — IndexedDB storage for key material, encrypted at rest using a key derived from the passkey challenge

**Local storage:** IndexedDB (encrypted via passkey-derived key). No private key material is ever transmitted to any server.

**Bundle IDs / identifiers:** Served from `auth.<your-domain>` via Cloudflare Workers.

## iOS / iPadOS (aegis-apple)

**Language:** Swift + SwiftUI
**Minimum iOS:** 14.0
**Minimum iPadOS:** 14.0
**Crypto:** Rust FFI via aegis-ffi (compiled to `.xcframework`)
**Bundle ID:** `VZMVM2W22Z.io.aegis.ios`

The iOS client is a native SwiftUI app sharing core logic with the macOS app via a **Swift Package** (`aegis-apple/shared/Package.swift`).

**Features:**
- Native Mail.app-like interface — inbox, compose, thread view
- Keychain-backed identity storage (encrypted at rest by the OS)
- iCloud sync via Core Data / CloudKit
- Background fetch for new messages
- Face ID / Touch ID vault unlock via Local Authentication framework
- Cross-device passkey sync via iCloud Keychain

**Shared Swift Package (`AegisAppKit`):**
- `InboxView`, `SetupView`, `ComposeView` — reusable SwiftUI components
- `Mailbox` — envelope fetching, decryption, local index
- `AppSession` — vault unlock, identity management, relay connection
- `PasskeyService` — biometric / PIN authentication
- `AegisFFI` — UniFFI-generated bindings to Rust (auto-generated by `uniffi-bindgen`)

**Compile targets:** `aarch64-apple-ios` (device) + `aarch64-apple-ios-sim` (simulator)

## Android (aegis-android)

**Language:** Kotlin + Jetpack Compose
**Minimum Android:** 7.0 (API level 24)
**Crypto:** Rust FFI via aegis-ffi (compiled to `.so`)
**Package:** `io.aegis.android`

**Features:**
- Native Compose UI — inbox, compose, identity, settings screens
- Android Keystore for hardware-backed key storage (software fallback on older devices)
- BiometricPrompt for fingerprint / face / PIN vault unlock
- Google Password Manager sync via Credential Manager API
- Room database for local envelope cache

**Compose screens:**
- `RootScreen` — onboarding or authenticated root
- `SetupScreen` — identity creation / import
- `MailScreen` — envelope inbox list
- `InboxScreen` — envelope detail, message decryption
- `SettingsScreen` — identity management, relay config, logout

**Compile targets:** `aarch64-linux-android` (ARM64), `armv7-linux-android` (ARMv7), `x86_64-linux-android` (emulator)

The Rust FFI library is built via a `cargoBuildAllArch` Gradle task. Kotlin bindings are auto-generated by `uniffi-bindgen` at build time.

## aegit CLI

**Language:** Rust
**Platform:** macOS, Linux, Windows (WSL)
**Install:** `cargo build --release` from [aegit-cli](https://github.com/mlaify/aegit-cli)

The CLI is the reference implementation of the AMP protocol — every feature of the protocol is exercisable from the command line.

**Identity commands:**
```bash
aegit id init [--alias ALIAS]           # create a new hybrid PQ identity
aegit id show [--identity ID]           # display identity document
aegit id list                           # list all local identities
aegit id publish --relay URL            # publish identity to relay
aegit id publish-prekeys --relay URL    # generate + publish one-time prekeys
```

**Message commands:**
```bash
aegit msg seal --to RECIPIENT --body TEXT [--relay URL] [--out FILE]
aegit msg open --input FILE
aegit msg list --recipient ID
```

**Relay commands:**
```bash
aegit relay push --relay URL --input FILE
aegit relay fetch --relay URL --recipient ID
aegit relay ack --relay URL --recipient ID --envelope-id UUID
aegit relay delete --relay URL --recipient ID --envelope-id UUID
aegit relay cleanup --relay URL
```

See [Getting Started](/aegis/getting-started/) for a full walkthrough.

## Shared cryptographic surface (aegis-ffi)

All native clients call the same FFI surface, generated by **UniFFI 0.28**:

```
generate_hybrid_pq_key_bundle() → String (JSON HybridPqKeyBundle)

seal_message(
  recipient_doc_json: String,
  sender_key_material_json: String,
  payload_json: String,
  sender_hint: Option<String>
) → String (JSON Envelope)

open_message(
  envelope_json: String,
  own_key_material_json: String,
  sender_doc_json: Option<String>
) → String (JSON PrivatePayload)

sign_identity_document(
  identity_doc_json: String,
  ed25519_seed_b64: String,
  ml_dsa_65_secret_b64: String
) → String (JSON signed IdentityDocument)

verify_identity_document(
  identity_doc_json: String
) → Result<(), FfiError>

claim_one_time_prekey_secret(
  claimed_prekey_json: String,
  own_prekey_secrets_json: String
) → String (JSON updated prekey secrets)
```

Swift bindings: `AegisFFI.swift` (auto-generated)
Kotlin bindings: `aegis_ffi.kt` (auto-generated)

## Admin panel (aegis-admin)

**Stack:** React 18 + Vite + TypeScript + Tailwind CSS
**Deployment:** Cloudflare Workers

A separate operator-facing console for managing relays and gateways. Not an end-user app.

**Capabilities:**
- Connect to any relay or gateway via URL + token
- Dashboard: envelope counts, active messages, identity counts, prekey pool levels
- Identity management: view published identities, claim and verify domains
- Prekey monitoring: pool levels per identity, bulk replenishment
- User provisioning: create users under a claimed domain
- Cleanup policy: configure retention (age, ack purge, expiry)
- Audit log viewer: browse the append-only JSONL audit trail
- Gateway: downgrade policy editor, SMTP relay config, user mapping

Source: [aegis-admin](https://github.com/mlaify/aegis-admin)
